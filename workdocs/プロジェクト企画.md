# 週5時間おとうさんプログラマー

## 0歳育児の困りごとを、小さなツールにしてみる実験室

### 最終企画案・運用設計・1年WBS

---

## 1. 企画の結論

このプロジェクトは、育児の正解を教えるメディアではない。

**子どもがまだ0歳の初めての父親が、育児で実際に困ったことを、週5時間だけ使って、小さなテンプレ・メモ・Webツールにして試していく実験ログ**である。

育児経験者として語らない。
専門家として語らない。
先輩パパとして教えない。
「うちではこう困ったので、こういう道具を作って試してみた」という記録だけを積み上げる。

最終的には、5〜10年後に **年200万円規模の副業収益** を目指す。ただし、1年目から大きく稼ぐのではなく、信用・実験ログ・テンプレ資産・小さな読者コミュニティを作ることを優先する。

---

## 2. 背景

この企画の背景には、3つの現実がある。

1つ目は、子どもが生まれ、0歳育児が始まったこと。
日々の育児は初めてのことだらけで、記録、予定、夫婦共有、買い物、予防接種、写真整理、離乳食準備など、小さな困りごとが大量に発生する。

2つ目は、AI時代におけるプログラマーとしての危機感。
純粋にコードを書くだけの価値は、今後相対的に下がっていく可能性がある。大手企業に属して安定収入を得ながらも、会社の外に自分の信用・発信・コミュニティ・商品を持つ必要がある。

3つ目は、すでに本業の給料上限がある程度見えていること。
大手企業に属して生活はできるが、将来の選択肢を増やすには、長期的に副業で収益を作る力が必要になる。

そのため、この企画は単なる育児ブログではない。
**育児 × プログラマー × AI時代の生存戦略 × コミュニティ形成** をつなげる長期プロジェクトである。

---

## 3. 最終コンセプト

### メインコピー

**週5時間おとうさんプログラマー**
**0歳育児の困りごとを、小さなツールにしてみる実験室**

### 説明文

育児の正解は語れない。
子どもはまだ0歳。
だからこれは、育児ノウハウではありません。

初めての育児で困ったことを、週5時間だけプログラマーとして小さな仕組みにしてみる実験ログです。

使えたものも、使えなかったものも記録します。
うちの家庭ではこうだった、という範囲だけを共有します。
専門的な判断は、医師・自治体・保育士・専門家・公式情報に任せます。

## 4. 立ち位置

この企画の立ち位置は、以下で固定する。

| 項目     | 内容                        |
| ------ | ------------------------- |
| 肩書き    | 週5時間おとうさんプログラマー           |
| 立場     | 初めて0歳育児をしている父親            |
| 役割     | 育児の困りごとを小さな道具にする実験者       |
| 語り方    | 「教える」ではなく「試した記録」          |
| 強み     | プログラマー視点で、家庭の困りごとを仕組みにできる |
| 弱み     | 育児経験は浅い。専門家ではない           |
| 信頼の作り方 | 分からないことは分からないと書く。失敗も書く    |

この企画は、経験豊富な育児インフルエンサーとは戦わない。
むしろ、経験が浅いことを隠さない。

**「初めての育児で困っている人が、同じ目線で見られる実験ログ」**にする。

## 5. 対象ユーザー

### メイン対象

**妊娠後期〜0歳〜1歳半くらいの子どもがいる、初めて育児中の親**

特に刺さるのは、以下のような人。

| 項目    | 内容                          |
| ----- | --------------------------- |
| 年齢    | 20代後半〜30代後半                 |
| 状態    | 初めての育児で、毎日分からないことだらけ        |
| 悩み    | 記録が散らばる、予定が不安、夫婦共有が難しい      |
| IT感覚  | スマホで使えるものなら試せる              |
| 欲しいもの | 正解ではなく、他の家庭のリアルな工夫          |
| 刺さる価値 | 「うちも同じことで困ってる」感             |
| 課金理由  | 実験ログを応援したい、テンプレや小ツールを使ってみたい |

### サブ対象

将来的には、以下の層も入ってくる。

| 層               | 内容                      |
| --------------- | ----------------------- |
| 育児中の会社員エンジニア    | AI時代のキャリア不安と育児の両方を抱えている |
| 親子向けサービスに興味がある人 | 育児と個人開発の過程を見たい          |
| これから父親・母親になる人   | 出産前後にどんな困りごとがあるか知りたい    |
| 個人開発に興味がある親     | 家庭内の困りごとをツール化する発想に興味がある |

ただし、初期の主語は広げすぎない。
最初の主対象はあくまで、**0歳前後の初めて育児中の親**である。

## 6. やらないこと

ここが最重要。

この企画は、以下をやらない。

| やらないこと         | 理由                |
| -------------- | ----------------- |
| 育児ノウハウを教える     | 経験・専門性がない         |
| 先輩パパとして語る      | 子どもがまだ0歳で、現在地とズレる |
| 片付け・学習・しつけを語る  | 実体験がない            |
| 「こうすべき」と断定する   | 家庭差・個人差が大きい       |
| 発達・睡眠・医療に踏み込む  | 専門領域であり危険         |
| 「効果がある」と言う     | 根拠が必要になり、誠実ではない   |
| 3〜8歳向け素材を中心にする | 今の実体験とズレる         |
| 子育ての正解を売る      | コンセプト崩壊           |
| 個別育児相談を受ける     | 責任が重すぎる           |
| 高額教材として売る      | 現段階の信用・実績と合わない    |

| やること    | 内容                           |

| 失敗ログ    | 微妙だった、続かなかった、家族に刺さらなかったことも書く |
| テンプレ    | 家庭内メモ、買い物表、予防接種確認表など         |
| 小ツール    | スマホで使える簡単な記録・確認ツール           |

| 無料公開    | ブログ・無料note・SNSで入口を作る         |
| 有料化     | 実験ログの裏側、テンプレ、小ツールDLを提供する     |
| 長期記録    | 子どもの成長に合わせてテーマも自然に変える        |

## 8. 発信・販売の全体構造

### 無料導線

| 場所        | 役割                        |
| --------- | ------------------------- |
| Threads   | 本音、育児あるある、実験中のメモ、失敗ログ     |
| Instagram | 実験ログの見える化、保存用カルーセル、テンプレ紹介 |
| 無料note    | 実験ログ記事、考え方、失敗談            |
| 個人ブログ     | 無料ツール、無料テンプレ、商品レビュー、検索流入  |
| リクエスト箱    | 困りごとデータの蓄積                |

### 有料導線

| 場所          | 役割                  |
| ----------- | ------------------- |
| noteメンバーシップ | 月額課金、実験ログの裏側、更新通知   |
| 個人ブログ会員ページ  | ID/PASSでテンプレ・小ツールDL |
| 有料note記事    | 今月のパスワード案内、DL導線     |
| 将来の講座・教材    | 2年目以降の収益拡張          |

noteメンバーシップは、開設・運用費用が無料で、メンバーシップのプラットフォーム利用料は10%と公式に説明されている。また、メンバーシップでは加入期間中に過去記事を含めた記事が読み放題になるため、実験ログの蓄積型と相性がよい。

また、noteのコンテンツ販売では、売上金額から事務手数料が引かれ、その残額にプラットフォーム利用料がかかる。クレジットカード決済の場合の事務手数料は5%とされているため、実際の手取りは額面より少なく見積もる必要がある。

いきなり月1,000円×300人は狙わない。

| フェーズ    | 時期      |          価格 | 目標                 |
| ------- | ------- | ----------: | ------------------ |
| Phase 1 | 1〜3ヶ月目  |          無料 | 反応を見る              |
| Phase 2 | 4〜6ヶ月目  |   月300〜500円 | 応援メンバーを試す          |
| Phase 3 | 7〜12ヶ月目 | 月500〜1,000円 | テンプレ・小ツールが溜まったら有料化 |

年200万円は、月約16.7万円。

| 収益源         | 例                  |

## 12. 自サイト設計

自サイトは、noteの補助ではなく、長期資産にする。
noteは課金と更新通知、自サイトは検索流入・ツール置き場・DLページとして使う。

### 最初から作るページ

| ページ     | 内容                     |
| ------- | ---------------------- |
| トップ     | 0歳育児の困りごとをツール化する実験室    |
| 実験ログ    | note記事の要約・ブログ版         |
| 無料テンプレ  | 一部無料DL                 |
| 無料ツール   | スマホで使える小ツール            |
| リクエスト箱  | 困りごと投稿フォーム             |
| 使ってみたもの | 買った/使った/使わなかった記録       |
| 会員限定DL  | ID/PASS付きページ           |
| 免責      | 育児・医療・発達アドバイスではないことを明記 |

### 会員限定DLページ

初期は高度な会員認証を作らない。
Basic認証または簡易パスワードで十分。

運用案：

* 月ごとにパスワードを変える
* noteメンバー限定記事または掲示板で案内する
* 重要データは置かない
* 再配布禁止を明記する
* 完全防御よりも、制作継続を優先する

## 13. 商品レビュー方針

育児用品レビューは、アフィリエイト感を強くしない。
「おすすめです」「買うべきです」ではなく、**うちではこうだった**に限定する。

書く項目：

* 買った理由
* 使った期間
* うちでは合った点
* うちでは合わなかった点
* 微妙だった点
* 次に買うならどうするか
* 使わなくなった理由

NG表現：

* 絶対おすすめ
* これを買えば解決
* 全家庭に必要
* 発達に良い
* 睡眠改善に効く
* 買わないと損

## 15. 記事テンプレ

毎回この型で書く。

1. 今週困ったこと
2. なぜ困ったか
3. 試しに作ったもの
4. 使ってみた結果
5. 微妙だった点
6. 改善するならどうするか
7. DLリンク・テンプレ案内
8. 専門的な話は公式・専門家へ
9. 次に試すこと
10. 困りごと募集

この型にすることで、「教えるnote」ではなく「実験ログ」になる。

## 21. リスクと対策

| リスク           | 対策                        |
| ------------- | ------------------------- |
| 育児アドバイス化してしまう | 記事テンプレに「うちでは」を必ず入れる       |
| 専門領域に踏み込む     | 医療・発達・睡眠・栄養は公式情報へのリンクに留める |
| 有料価値が弱い       | 最初は300〜500円の応援枠にする        |
| 継続できない        | 週5時間を上限にし、縮小運用を許可する       |
| 家族に負担がかかる     | 家庭内の実験は無理に巻き込まない          |
| アフィリエイト臭が出る   | 「おすすめ」ではなく「うちではこうだった」に限定  |
| ネタ切れ          | リクエスト箱と実生活の困りごとから拾う       |
| 収益化を焦る        | 1年目は信用・ログ・資産作りに固定する       |

## 25. 最終方針

このプロジェクトは、育児の先生になる企画ではない。

最終方針はこれである。

**初めての0歳育児で困ったことを、
週5時間だけ使って、
小さなテンプレ・メモ・ツールにしてみる。
使えたものも、使えなかったものも記録する。**

1年目は、信用と実験ログ作り。
2年目で、月1,000円×300人を狙う。
5〜10年後に、年200万円規模の副業収益を目指す。

この順番が最も自然で、嘘が入りにくく、現在地と矛盾しない。
